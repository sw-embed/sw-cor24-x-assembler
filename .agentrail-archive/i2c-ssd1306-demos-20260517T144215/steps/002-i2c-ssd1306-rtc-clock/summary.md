Both SSD1306 demos shipped. Hello: minimal init + 25-byte HELLO write, halts in 6491 instructions. Clock: combines DS1307 read + SSD1306 draw, renders HH:MM:SS at page 0 col 0, redraws each ~10k spin iterations, loops forever (non_halting). Both registered in examples() alphabetically (I2C OLED Hello, I2C OLED RTC Clock between I2C Add1 Ping and I2C RTC Read); clock in non_halting. Verified via cor24-emu --i2c-device ssd1306@0x3C [+ ds1307@0x68?hour=...&minute=...&second=...] with --dump-i2c: init bursts + data writes all ACK'd, no protocol errors. Hit one mul-related bug along the way (mul r0, r2 in render_digit produced wrong nibble offset; sidestepped with explicit add chain; worth a separate debug session).
